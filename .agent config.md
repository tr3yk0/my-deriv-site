## Overview

Deriv Bot is a web-based automated trading platform that lets users create trading bots **without writing a single line of code**. Using a visual block-based programming interface (powered by [Blockly](https://developers.google.com/blockly)), users drag and drop logic blocks to define trading strategies, then run them live against Deriv's real-time markets.

Users can:
- Build bots from scratch using the block editor
- Load one of **12 pre-built Free Bot templates** (Speed Trading, AI Trading, Pattern Analysis, and more)
- Import/export bot configurations as XML files
- Trade on **demo or real accounts** via the Deriv WebSocket API
- Sync bot strategies to **Google Drive**

## Architecture

### State Management

State is managed with **MobX** using a root store pattern. All domain-specific stores live in `src/stores/` and are aggregated by a single root store that is injected via React context.

```
RootStore
├── ClientStore       — account, session, auth state
├── DashboardStore    — bot builder UI state
├── ChartStore        — charting and market data state
├── RunPanelStore     — bot run/stop/status state
└── ...               — other domain stores
```

**Rule:** Always check the relevant store before adding new state elsewhere. Do not create ad-hoc local state for data that belongs in the store layer.

### Visual Bot Builder

The bot builder is powered by **Blockly**. Users connect visual blocks that map to trading operations (buy/sell, conditions, loops, math, etc.).

- Custom block definitions live in `src/blocks/`
- Toolbox (the block palette) is configured in `src/toolbox/`
- Bot strategies are serialized to **XML** for saving, loading, and sharing
- Pre-built bot templates are stored as XML files in `public/bots/`

> ⚠️ **Important:** Changes to block definitions or toolbox configs can silently break existing user-saved workspaces. Always test workspace serialization after any Blockly modification.

### Trading API

All trading operations go through **`@deriv/deriv-api`** using a persistent WebSocket connection.

- Real-time market data streaming
- Order execution (buy/sell)
- Account balance and portfolio updates
- Support for demo accounts, real accounts, and wallet-based accounts

> ⚠️ **Never** make raw HTTP calls directly to Deriv endpoints. Always use the existing service layer in `src/services/`.

### Authentication

Authentication uses **OAuth2 / OIDC** via `@deriv-com/auth-client`, backed by a **Token Management Backend (TMB)** for enhanced session handling.

- Multi-account support with seamless account switching
- Demo and real account types
- Wallet-based account support

> Do not build custom auth flows. All authentication must go through `@deriv-com/auth-client`.

- Project-specific rules (Blockly serialization safety, Deriv API service layer, i18n requirements, etc.)

## Contributing

1. Fork the repository and create a feature branch from `main`.
2. Read `ARCHITECTURE.md` and `.agent-config.toml` before making changes.
3. Follow the coding rules in `.agent-config.toml → [ai.instructions]`.
4. All user-visible strings **must** use `@deriv-com/translations` — no hardcoded text.
5. New UI features **must** be gated behind a GrowthBook feature flag.
6. Run the full CI pipeline before opening a PR:
   ```bash
   npm run build && npm run test -- --coverage && npm run lint && npm run typecheck
   ```
7. Do **not** modify `public/bots/` XML files without explicit sign-off — these are live user assets.
